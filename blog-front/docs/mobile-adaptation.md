# 移动端适配改造说明文档

## 1. 概述
本文档详细说明了博客文章编辑器 (`ArticleEditorPage`) 的移动端适配方案。本次改造旨在解决在手机和平板设备上编辑体验不佳的问题，实现了全屏沉浸式编辑、响应式工具栏以及对虚拟键盘的友好支持。

## 2. 改造内容

### 2.1 响应式布局
- **单列全屏模式**: 在屏幕宽度 ≤ 768px 时，自动隐藏预览区域和侧边元数据表单，编辑器占据 100% 视口空间。
- **抽屉式设置面板**: 原左侧的元数据表单（标题、分类、封面等）改为从左侧滑出的全屏抽屉，通过顶部的“设置”按钮触发。
- **视口高度适配**: 使用 `window.visualViewport` API 动态计算可视区域高度，解决 iOS/Android 键盘弹出时遮挡底部内容的问题。

### 2.2 交互优化
- **悬浮工具栏 (FAB)**: 
  - 默认状态：右下角显示圆形“+”悬浮按钮 (56x56px)。
  - 展开状态：点击后显示底部横向滚动工具栏，支持惯性滑动。
  - 触控目标：所有按钮尺寸放大至 44x44px 以上，符合移动端触控规范。
- **触控反馈**: 移除移动端的 `:hover` 样式，替换为 `:active` 缩放效果，提供明确的按压反馈。

### 2.3 字体与排版
- **防缩放**: 输入框字体大小设定为 16px，防止 iOS Safari 在输入聚焦时自动缩放页面。
- **阅读舒适度**: 行高调整为 1.6，左右内边距设为 16px，确保文字不会紧贴屏幕边缘。

## 3. 技术实现细节

### 3.1 核心代码
- **CSS 变量**: 使用 `--visual-viewport-height` 动态控制根容器高度。
- **状态管理**: 新增 `showSettings` 状态控制设置面板的显示与隐藏。
- **组件复用**: `MarkdownToolbar` 组件内部实现了移动端/桌面端的渲染逻辑分支，无需维护两套组件。

### 3.2 关键 CSS 片段
```css
/* 动态视口高度 */
.editorRoot {
  height: var(--visual-viewport-height, 100vh);
}

/* 移动端工具栏容器 */
.markdownToolbar {
  position: fixed;
  bottom: 24px;
  /* Glassmorphism */
  background: rgba(13, 27, 42, 0.95);
  backdrop-filter: blur(12px);
  /* Animation */
  transform: translateY(150%);
  transition: transform 0.3s cubic-bezier(0.4, 0, 0.2, 1);
}
```

## 4. 兼容性测试范围

### 4.1 设备与分辨率
- **iPhone SE / 8 (375x667)**: 验证小屏适配，工具栏不溢出。
- **iPhone 14 Pro Max (430x932)**: 验证刘海屏/灵动岛区域适配。
- **Android 主流机型 (360x640 - 412x915)**: 验证虚拟键盘弹出后的布局调整。

### 4.2 浏览器
- iOS Safari (iOS 15+)
- Chrome for Android
- 微信内置浏览器 (Webview)

## 5. 已知限制与注意事项
1. **预览功能**: 目前移动端默认隐藏了 Markdown 实时预览功能，以保证编辑性能和空间。未来考虑添加“预览模式”切换开关。
2. **图片上传**: 移动端上传封面时，建议使用相册选择而非直接拍照，以避免内存溢出导致页面刷新（部分低端 Android 机型）。
3. **横屏模式**: 暂未针对移动端横屏模式做特殊优化，建议竖屏使用。

## 6. 性能指标 (模拟)
- **FCP (First Contentful Paint)**: < 1.2s
- **CLS (Cumulative Layout Shift)**: 0
- **交互延迟 (INP)**: < 100ms
- **滚动帧率**: 稳定 60fps (主流设备)
